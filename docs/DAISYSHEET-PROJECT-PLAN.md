ExcelDNA provides `ExcelAsyncUtil.Run` which uses the RTD (Real-Time Data) mechanism internally.\n\n**File: `Async/AsyncFormulaHelper.cs`**\n\nAlgorithm:\n1. Formula is called synchronously by Excel\n2. Build a cache key from function name + all arguments\n3. Check `LocalResultCache` -- if hit and not expired, return immediately\n4. If miss, call `ExcelAsyncUtil.Run(cacheKey, async delegate)` which returns `#GETTING_DATA` placeholder\n5. The async delegate calls the DaisyReport API via `HttpClient`\n6. On completion, ExcelDNA triggers a recalculation of the cell\n7. Formula re-enters, finds result in memory cache, returns value\n8. Simultaneously, write result to SQLite cache for persistence across sessions\n\n**File: `Async/FormulaObservable.cs`**\n\nFor auto-refresh scenarios, implement `IExcelObservable`:\n- Subscribe to WebSocket messages from the server\n- When a \"data-changed\" notification arrives for a relevant datasource, trigger recalculation\n- Timer-based refresh also uses this mechanism (configurable interval, default 5 minutes)\n\n### 3.4 Local Result Cache\n\n**File: `Cache/LocalResultCache.cs`**\n\nSQLite database stored at `%APPDATA%\\DaisySheet\\cache.db`.\n\nSchema:\n```sql\nCREATE TABLE ds_cache (\n    cache_key     TEXT PRIMARY KEY,\n    result_json   TEXT NOT NULL,\n    created_at    TEXT NOT NULL,\n    expires_at    TEXT NOT NULL,\n    query_hash    TEXT,\n    connection_id TEXT\n);\nCREATE INDEX idx_cache_expires ON ds_cache(expires_at);\n```\n\nThe cache key is computed as `SHA256(functionName + \"||\" + String.Join(\"||\", allArgs))`. Default TTL is 300 seconds (configurable). On workbook open, stale entries are purged asynchronously.\n\n### 3.5 Authentication & Token Management\n\n**File: `Auth/TokenManager.cs`**\n\nAlgorithm:\n1. On first formula call or add-in load, check `%APPDATA%\\DaisySheet\\auth.json` for stored token\n2. Decode JWT without validation to check `exp` claim\n3. If expired or missing, show `LoginDialog` (WPF window)\n4. Login dialog posts to `POST /api/auth/login` with username/password\n5. Store JWT and refresh token in `auth.json` (encrypted with DPAPI on Windows)\n6. All subsequent HTTP calls include `Authorization: Bearer <token>` header\n7. On 401 response, attempt refresh; if refresh fails, show login dialog again\n8. Support Windows Integrated Auth: if server supports it, send NTLM/Negotiate token automatically\n\n`auth.json` structure (DPAPI-encrypted):\n```json\n{\n  \"serverUrl\": \"https://daisyreport.company.com\",\n  \"token\": \"eyJ...\",\n  \"refreshToken\": \"...\",\n  \"expiresAt\": \"2026-03-30T12:00:00Z\",\n  \"username\": \"john.doe\"\n}\n```\n\n### 3.6 Ribbon UI\n\n**File: `Ribbon/DaisySheetRibbon.cs`** (extends `ExcelRibbon`)\n**File: `Ribbon/DaisySheetRibbon.xml`**\n\nRibbon tab \"DaisySheet\" with groups:\n- **Connection**: Connect, Disconnect, Connection Manager (dropdown of saved connections)\n- **Query**: Insert Formula (wizard), Query Designer (launches WPF app), Browse Queries\n- **Data**: Refresh Selected, Refresh All, Auto-Refresh toggle, Clear Cache\n- **Tools**: Settings, Drilldown Mode toggle, About/Help\n- **Status**: Connection status indicator (green/yellow/red), logged-in user display\n\nEach ribbon button has a `getEnabled` callback that checks `TokenManager.IsAuthenticated`. Phase 4: GL/ERP Formula Engine\n\n**Estimated complexity: HIGH (6-8 weeks)**\n**Dependencies: Phase 2 (Data Providers)**\n\n### 6.1 GL Formula Architecture\n\nThe GL engine is the most complex component because each ERP system stores its chart of accounts and GL balances differently. The engine uses an adapter pattern with a `GenericSqlConnector` fallback.\n\n**File: `DaisySheet.FormulaEngine/GlEngine/GlFormulaProcessor.cs`**\n\nCentral orchestrator:\n1. Receive `GlAccountRequest` (connection, account segments, period, year, balance type)\n2. Look up the connection's ERP type from `RS_DATASOURCE` metadata\n3. Route to the appropriate `IErpConnector` implementation\n4. The connector generates the correct SQL for that ERP's GL tables\n5. Execute via `IDataProvider`\n6. Return the balance as a decimal\n\n### 6.2 Account Segment Parsing\n\n**File: `GlEngine/AccountSegmentParser.cs`**\n\nGL account numbers in ERPs use segmented structures. Examples:\n- Simple: `1000` (single segment)\n- SAP: `1000-100-CC01` (company code-account-cost center)\n- Oracle EBS: `01-110-1000-0000-000` (company-department-account-subaccount-project)\n- Dynamics GP: `000-1100-00` (segment1-segment2-segment3)\n\nThe parser takes a format string (configurable per connection) and splits the account string. Format string example: `{Company}-{Account}-{CostCenter}` where each segment maps to a database column.\n\n### 6.3 Period Resolution\n\n**File: `GlEngine/PeriodResolver.cs`**\n\nERP GL periods do not always map to calendar months. The resolver:\n1. Reads fiscal calendar from the ERP (configurable SQL query per ERP type)\n2. Maps \"Period 1\" to the actual date range\n3. Supports special periods: 0 (beginning balance), 13/14 (year-end adjustments)\n4. Supports relative references: `CUR` (current period), `CUR-1` (prior period), `YTD` (year-to-date sum)\n\n### 6.4 Balance Type Resolution\n\n**File: `GlEngine/BalanceTypeResolver.cs`**\n\nBalance types determine how periods are aggregated:\n- `PTD` (Period-to-Date): Just the specified period\n- `YTD` (Year-to-Date): Sum of period 1 through the specified period\n- `QTD` (Quarter-to-Date): Sum from quarter start to the specified period\n- `MTD` (Month-to-Date): Same as PTD for monthly periods\n- `BAL` (Balance): Beginning balance + YTD activity\n- `BEG` (Beginning): Opening balance of the year\n\nThe resolver generates different SQL WHERE clauses for each type.\n\n### 6.5 Generic SQL Connector\n\n**File: `Erp/GenericSqlConnector.cs`**\n\nFor ERPs without a dedicated connector, the user configures:\n1. **GL Balance Query Template**: SQL with placeholders like `{Account}`, `{Period}`, `{Year}`\n2. **GL Detail Query Template**: SQL that returns transaction details\n3. **Account Table**: Which table/view contains the chart of accounts\n4. **Period Table**: Which table/view contains the fiscal calendar\n\nThese templates are stored in `RS_ERP_CONNECTOR_CONFIG` (new table, see Section 13).\n\n### 6.6 ERP Connector Interface\n\n**File: `Erp/IErpConnector.cs`**\n\n```csharp\npublic interface IErpConnector\n{\n    string ErpName { get; }\n    Task<decimal> GetAccountBalanceAsync(IDbConnection conn, GlAccountRequest request);\n    Task<DataTable> GetAccountDetailAsync(IDbConnection conn, GlAccountRequest request);\n    Task<DataTable> GetAccountRangeAsync(IDbConnection conn, string accountFrom, string accountTo,\n        int period, int year, string balanceType);\n    Task<IReadOnlyList<GlAccount>> GetChartOfAccountsAsync(IDbConnection conn);\n    Task<IReadOnlyList<FiscalPeriod>> GetFiscalCalendarAsync(IDbConnection conn, int year);\n}\n```\n\nPhase 4 MVP delivers `GenericSqlConnector` only.
